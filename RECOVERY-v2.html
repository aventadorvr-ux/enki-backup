📋 PART 1: DISASTER RECOVERY (Complete Server Loss)

Prerequisites

  • New Ubuntu/Debian server (or fresh EC2 instance)
  • Internet connection
  • GitHub access (for cloning backup)
  • Telegram app (to reconnect with Enki)
1 Install OpenClaw 
npm install -g openclaw
openclaw --version
2 Clone Backup Repository 
git clone https://github.com/aventadorvr-ux/enki-backup.git \
  ~/.openclaw/workspace
cd ~/.openclaw/workspace
npm install
3 Install System Dependencies 
sudo apt-get update
sudo apt-get install -y chromium-browser fail2ban
4 Test JavaScript Scraper 
node js-fetch.js "https://www.trademe.co.nz" test.json
cat test.json | head -20
rm test.json
5 Reconnect Telegram 
openclaw pairing list
openclaw pairing add telegram --label "Enki Restored"
# Approve on your phone, then message the bot
6 Restore Security Hardening 
# Enable UFW Firewall
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp comment 'SSH access'
sudo ufw limit 22/tcp comment 'Rate limit SSH'
sudo ufw enable

# Harden SSH
sudo tee /etc/ssh/sshd_config.d/hardening.conf << 'EOF'
PasswordAuthentication no
PermitRootLogin no
X11Forwarding no
MaxAuthTries 3
AllowUsers ubuntu
EOF
sudo systemctl reload ssh

# Start fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

🔌 PART 2: CONNECTION RECOVERY (OpenClaw/Telegram Issues)

Use Case: Connection lost due to token/credit issues, Telegram disconnect, or process failure.

When to Use This Guide

  • Ran out of OpenRouter credits and topped up
  • Telegram connection lost
  • OpenClaw not responding to messages
  • Process crashed or was killed
1 SSH into EC2 Instance 
ssh -i your-key.pem ubuntu@<EC2_PUBLIC_IP>
2 Check OpenClaw Process 
ps aux | grep openclaw

Observe PID for openclaw-gateway. Check disk space:

df -h

Note: If disk is almost full, proceed to Step 3.

3 Clear Disk Space (if needed) 
sudo journalctl --vacuum-time=2d
sudo apt clean
npm cache clean --force

Ensure at least ~1GB free space.

4 Inspect OpenClaw Directory 
cd ~/.openclaw
ls -la

Key folders: telegram/, logs/, delivery-queue/, credentials/
Key file: openclaw.json

5 Stop Any Running OpenClaw Process 
sudo kill <PID>
6 Backup and Reset Telegram Session 
mv telegram telegram_backup
7 (Optional) Clear Stuck Delivery Queue 
mv delivery-queue delivery-queue_backup
mkdir delivery-queue
8 Restart OpenClaw 
openclaw

Look for startup message:

"I'll do the boring stuff while you dramatically stare at the logs like it's cinema. OpenClaw 2026.2.26"
9 Approve Telegram Pairing

A new pairing code will be printed in the logs.

openclaw pairing approve telegram <PAIRING_CODE>

Test in Telegram: Send /start

OpenClaw should respond → connection restored.

⚠️ Critical Notes

  • Do NOT use openclaw start (unknown command)
  • Old Telegram session can break if OpenRouter credits run out
  • Disk space must remain adequate for logs and session storage
  • Recommended: Monitor EC2 disk usage regularly
  • Optional: Use PM2 or Docker for auto-restart

✅ VERIFICATION CHECKLIST

After any restore or reconnection:

  • ☐ OpenClaw installed and responding
  • ☐ js-fetch.js works on test URL
  • ☐ Can message Enki via Telegram
  • ☐ All memory files present
  • ☐ Cron jobs scheduled
  • ☐ Firewall active (sudo ufw status)
  • ☐ Fail2ban running (sudo systemctl status fail2ban)
  • ☐ Git push works (optional)
